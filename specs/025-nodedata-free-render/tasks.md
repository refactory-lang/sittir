## Phase 1: Foundational — render_with_trivia! macro

- [ ] T001 Define `render_with_trivia!` macro in `rust/crates/sittir-core/src/types.rs` (or a new `macros.rs`) — reads `self.transport_trivia_data`, writes leading text to dest, calls render, writes trailing text to dest. Handles `None` gracefully.
- [ ] T002 Add `cargo test` for the macro in `rust/crates/sittir-core/src/engine.rs` — verify leading/trailing trivia streams correctly, verify no-op when trivia is None.

---

## Phase 2: US1 — Nested trivia via macro (Priority: P1)

**Goal**: Every `RenderableTransport::render_into` impl wraps with `render_with_trivia!`.

- [ ] T003 [US1] Update emitter (`packages/codegen/src/emitters/render-module.ts`) to emit `render_with_trivia!(self, dest, render_xxx_transport(self, dest))` in every `RenderableTransport::render_into` impl instead of bare `render_xxx_transport(self, dest)`.
- [ ] T004 [US1] Regen all grammars + `cargo check` — zero errors. Verify nested trivia test (attach trivia to a child node, render parent, child trivia appears).
- [ ] T005 [US1] Add integration test: factory node with trivia on nested child renders correctly through native engine.

## Phase 3: US2 — Format without NodeData (Priority: P1)

**Goal**: `render_canonical_node` replaced by `apply_render_format(kind_id, span, canonical, format)`.

- [ ] T006 [US2] Refactor `render_canonical_node` in `rust/crates/sittir-core/src/engine.rs` to `apply_render_format(kind_id: KindId, span: Option<&Span>, canonical: String, format: Option<&FormatRecord>) -> String`. Keep as standalone function, not method on ParsedTree.
- [ ] T007 [US2] Update `ParsedTree::render_node_data` to call `apply_render_format` with `node.type_` and `node.span`.
- [ ] T008 [US2] Update napi entry point in `lib.rs` (generated by render-module.ts) — read `$type` (KindId) and `$span` from JS object before transport construction, pass to `apply_render_format` after `render_transport_dispatch`.
- [ ] T009 [US2] Regen + `cargo check` + verify format still applies correctly for parsed nodes.

---

## Phase 4: US3 — Eliminate render_dispatch + templates.rs functions (Priority: P1)

**Goal**: One render path — transport only. NodeData render uses streaming inline.

- [ ] T010 [US3] In emitter, stop emitting per-kind `render_xxx(node: &NodeData) -> Result<String>` functions in templates.ts output. Templates.ts becomes struct definitions only.
- [ ] T011 [US3] In emitter, stop emitting `render_dispatch` match table in `dispatch.rs`. Delete `dispatch.rs` output entirely.
- [ ] T012 [US3] Emit `render_nodedata_into(node: &NodeData, dest: &mut dyn Write) -> Result<()>` in `rust/crates/sittir-{lang}/src/render/bridge.rs` — constructs template from NodeData fields and calls `write_into(dest)`. This replaces `render_dispatch`.
- [ ] T013 [US3] Update `EngineGrammar::render` trait impl to call `render_nodedata_into` with a String buffer instead of `render_dispatch`.
- [ ] T014 [US3] Regen all grammars + `cargo build --release` — zero warnings, zero errors. Validator counts hold (native backend).

---

## Phase 5: US4 — Remove node_data_from_transport (Priority: P2)

**Goal**: Dead code cleanup.

- [ ] T015 [US4] In emitter, stop emitting `node_data_from_transport` in `bridge.rs`. Remove the function entirely from generated output.
- [ ] T016 [US4] Update `render_transport_parts` — no longer calls `node_data_from_transport`. Instead, read KindId + Span from the transport directly (emitter adds accessors or reads from napi pre-construction).
- [ ] T017 [US4] Regen + verify no references to `node_data_from_transport` remain. `cargo check` clean.

## Implementation Strategy

**MVP**: Phases 1-2 (T001-T005). Nested trivia via macro — streaming, no buffer for trivia. Immediate value.

**Core**: Phase 3-4 (T006-T014). Eliminate NodeData from transport, kill render_dispatch. The architectural win.

**Cleanup**: Phase 5-6 (T015-T021). Dead code removal + verification.

**Suggested parallel execution**:
- T001 + T006 (macro in core + format refactor in engine — different files)
- T010 + T011 (stop emitting templates fns + stop emitting dispatch — same emitter, different sections)
