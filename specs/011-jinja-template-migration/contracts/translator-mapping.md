# Contract: YAML-to-Jinja Translator Mapping

The deterministic transformation from the existing `AssembledNode`
render metadata (and current YAML output) to per-rule `.jinja` files.
Implemented in `packages/codegen/src/emitters/jinja-translator.ts`.

## Input → Output summary

| YAML-era construct | `.jinja` output | Notes |
|--------------------|-----------------|-------|
| `kind: "$NAME_PLACEHOLDER_TEMPLATE"` (bare string) | Direct translation of `$VAR` → `{{ var }}` | Most rules post ADR-0013 Task 2 collapse |
| `kind: { template: "...", joinBy: "X" }` | Single `.jinja` file; `joinBy` applied in `prepare()` (pre-joined into `children`) | Template itself uses `{{ children }}` |
| `kind: { template: "...", <field>_clause: "..." }` | Template + `{% if <field> %}...{% endif %}` | Clause bodies inlined into the template |
| `kind: { variants: { form_a: "T_a", form_b: "T_b" } }` | For the 5 rules that genuinely branch, emit `{% if variant == "form_a" %}...{% elif variant == "form_b" %}...{% endif %}`. For collapsed-identical (post ADR-0013 Task 2), emit the single body. | |
| `$NAME` | `{{ name }}` | snake_case preserved |
| `$$$NAME` (multi-valued, pre-joined) | `{{ name }}` | `joinBy` already applied by `prepare()` |
| `$$$CHILDREN` | `{{ children }}` | |
| `$$$NAME` (per-item iteration needed, rare) | `{% for item in name_list %}{{ item }}{% if not loop.last %}<sep>{% endif %}{% endfor %}` | Reserved for edge cases where the per-joined string doesn't capture intent |
| `$TEXT` | `{{ text }}` | |
| `$NEWLINE` | `\n` literal | Emitted as a literal newline in the `.jinja` file |
| `$INDENT`, `$DEDENT` | Empty string (column tracking handles) | Stripped — render-time indent logic unchanged |

## Detailed mapping rules

### Rule 1: Single-template branch / container

**Input** (post collapse):
```yaml
struct_item: $VISIBILITY_MODIFIER struct $NAME $TYPE_PARAMETERS $$$CHILDREN
```

**Output** (`struct_item.jinja`):
```jinja
{# @generated by @sittir/codegen — do not edit. #}
{{ visibility_modifier }} struct {{ name }}{{ type_parameters }}{{ children }}
```

Rationale: `$VAR` → `{{ var }}`, one-to-one. `$$$CHILDREN` → `{{ children }}` since `prepare()` pre-joins.

### Rule 2: Clause-bearing template

**Input**:
```yaml
function_item:
  template: fn $NAME $PARAMETERS $RETURN_TYPE_CLAUSE $BODY_CLAUSE
  return_type_clause: -> $RETURN_TYPE
  body_clause: $BODY
```

**Output** (`function_item.jinja`):
```jinja
{# @generated by @sittir/codegen — do not edit. #}
fn {{ name }} {{ parameters }}
{%- if return_type %} -> {{ return_type }}{% endif -%}
{%- if body %} {{ body }}{% endif -%}
```

Rationale: Clauses become `{% if <field> %}<body>{% endif %}`. The body itself undergoes the same `$VAR` → `{{ var }}` translation. Whitespace control (`{%-`, `-%}`) replaces the YAML-era implicit space absorption around absent clauses.

### Rule 3: Variant-branching rule (5 total)

**Input**:
```yaml
visibility_modifier:
  variants:
    form0: $$$CHILDREN
    form1: $PUB($$$CHILDREN$IN)
```

**Output** (`visibility_modifier.jinja`):
```jinja
{# @generated by @sittir/codegen — do not edit. #}
{%- if variant == "form0" -%}
{{ children }}
{%- elif variant == "form1" -%}
{{ pub }}({{ children }}{{ in }})
{%- endif -%}
```

Rationale: Explicit `variant` comparison. Whitespace control strips newlines between branches. Matches the render behavior that `obj.variants[node.$variant]` produced in the regex substitutor.

### Rule 4: Leaf (text-only) rule

**Input**: rule has a `PatternRule` body; codegen does not emit a template entry (leaves render via `$text`).

**Output**: No `.jinja` file. Renderer's token-shaped-kind path handles leaf rendering (unchanged from ADR-0013 Task 1).

### Rule 5: Keyword / token rule

**Input**: rule is an `AssembledKeyword` or `AssembledToken` — no template entry in YAML today.

**Output**: No `.jinja` file. Renderer inlines the keyword's literal text at the call site (via the parent's template, not via a dedicated render path).

### Rule 6: Supertype / enum / group / multi

**Input**: rule is a dispatch point; no template body.

**Output**: No `.jinja` file at the supertype level. Forms / subtypes each have their own template file.

## Invariants the translator MUST enforce

1. **Deterministic output**: Re-running the translator on the same `NodeMap` produces byte-identical `.jinja` files. No timestamps, no random ordering.
2. **Loud failure**: When a rule's metadata implies a construct outside the authoring subset, the translator throws with the rule kind and the specific offending construct. Never emits a degraded template.
3. **Directory consistency**: Before emission, the translator deletes any `.jinja` files in `packages/<grammar>/templates/` whose filenames don't correspond to a current rule in the `NodeMap`. Prevents stale files from drift across regenerations.
4. **Comment header**: Every emitted file begins with the `@generated` comment.
5. **Whitespace fidelity**: Trailing newlines in YAML block scalars are preserved; leading/trailing spaces around `$VAR` placeholders become literal spaces in the `.jinja` template, with `{%-`/`-%}` added only where the YAML semantics required absorption.

## Test strategy

The translator lives under `packages/codegen/src/emitters/jinja-translator.ts`. Its unit tests (`packages/codegen/src/__tests__/jinja-translator.test.ts`) exercise one case per mapping rule above, with fixture `AssembledNode` inputs and expected `.jinja` text outputs. Byte-equality on the output file text is the gate.

Integration gate: post-emission, the full round-trip corpus runs with the Nunjucks renderer against the generated `.jinja` files and must produce byte-identical output to the regex-substitutor baseline.
