# Data Model: Rewrite Codegen from grammar.js

**Feature**: 001-codegen-grammarjs-rewrite
**Date**: 2026-03-24

## Input Entities (read-only, from grammar packages)

### GrammarJson
Source: `grammar.json` (compiled from `grammar.js`)
- `name: string` — grammar language name (e.g., "rust")
- `rules: Record<string, GrammarRule>` — rule definitions per node kind

### GrammarRule (discriminated union)
Types: SEQ, CHOICE, STRING, FIELD, SYMBOL, BLANK, REPEAT, REPEAT1,
PREC, PREC_LEFT, PREC_RIGHT, ALIAS, TOKEN, IMMEDIATE_TOKEN, PATTERN

Key relationships:
- SEQ.members → ordered list of child rules (determines token order)
- FIELD.name → field name, FIELD.content → nested rule
- STRING.value → literal text (keywords, punctuation)
- SYMBOL.name → reference to another rule

### RawNodeEntry
Source: `node-types.json`
- `type: string` — node kind string
- `named: boolean` — whether this is a named node
- `fields?: Record<string, RawFieldEntry>` — field metadata
- `children?: RawFieldEntry` — unnamed children slot
- `subtypes?: Array<{ type: string; named: boolean }>` — for supertypes

### RawFieldEntry
- `required: boolean` — whether field must be present
- `multiple: boolean` — whether field accepts multiple values
- `types: Array<{ type: string; named: boolean }>` — accepted node kinds

## Derived Entities (computed by grammar-reader)

### NodeMeta
Computed by: `readGrammarNode(grammar, kind)`
Merges `node-types.json` (field metadata) + `grammar.json` (rule structure)
- `kind: string` — node kind
- `fields: FieldMeta[]` — field metadata
- `hasChildren: boolean` — whether node has unnamed children
- `children?: ChildrenMeta` — unnamed children metadata

### FieldMeta
- `name: string` — field name (from grammar)
- `required: boolean` — from node-types.json (authoritative)
- `multiple: boolean` — from node-types.json (authoritative)
- `types: string[]` — all accepted type strings
- `namedTypes: string[]` — only named types (maps to builders)

### ChildrenMeta
- `required: boolean`
- `multiple: boolean`
- `types: string[]`
- `namedTypes: string[]`

## Output Entities (generated by codegen)

### Builder (base class, in @sittir/types)
Abstract class extended by all generated builders.
- `renderImpl(ctx?): string` — render to source text
- `build(ctx?): N` — build plain-object IR node
- `toCST(offset?, ctx?): CSTNode` — build CST node tree
- `toEdit(startPos, endPos): Edit` — create byte-range replacement
- `render(validate?, ctx?): string | Promise<string>` — render with validation
- `toCSTChildren(ctx?): CSTChild[]` — structured children for CST
- `nodeKind: string` — the node kind (getter)

### Edit (interface, in @sittir/types)
- `startPos: number` — byte offset of replacement start
- `endPos: number` — byte offset of replacement end
- `insertedText: string` — replacement text

### RenderContext (interface, in @sittir/types)
- `parser?: unknown` — tree-sitter parser for full validation
- `indent?: string` — indentation unit
- `format?: (source: string) => string | Promise<string>` — external formatting hook

### GeneratedFiles (returned by generate())
- `grammar: string` — grammar.ts source
- `types: string` — types.ts source
- `builders: Map<string, string>` — kind → builder .ts source
- `builder: string` — builder.ts (ir namespace) source
- `consts: string` — consts.ts source
- `tests: Map<string, string>` — kind → test .ts source
- `leafTests: string` — terminal builder regression tests
- `config: string` — vitest.config.ts source
- `index: string` — barrel index.ts source
