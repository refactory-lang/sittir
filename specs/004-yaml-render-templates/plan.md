# Implementation Plan: YAML Render Templates

**Branch**: `004-yaml-render-templates` | **Date**: 2026-03-30 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `specs/004-yaml-render-templates/spec.md`
**Design Document**: [design.md](design.md) — render engine pseudocode, type changes, full Rust YAML example

## Summary

Migrate sittir's render template format from S-expression strings in TypeScript (`rules.ts` + `joinby.ts`) to ast-grep-style YAML templates (`templates.yaml`) with synthesized clauses for optional token+field groups, per-rule `joinBy` separators, and literal formatting. This resolves 5 known open issues (#1, #5, #7, #8, #9), eliminates the S-expression parser, and reduces the render engine from ~250 lines to ~50 lines.

## Technical Context

**Language/Version**: TypeScript (ESM, `.ts` extensions in imports), TypeScript 6.0.2
**Primary Dependencies**: `@sittir/core`, `@sittir/types`, `@sittir/codegen`; tree-sitter grammars (grammar.json + node-types.json)
**Storage**: File system (YAML templates at `packages/{lang}/templates.yaml`, read at codegen time)
**Testing**: Vitest
**Target Platform**: Node.js (library consumed by codemod scripts and tooling)
**Project Type**: Library (monorepo with pnpm workspaces — 7 packages)
**Performance Goals**: N/A (build-time codegen tool, not latency-sensitive)
**Constraints**: Zero runtime dependencies for generated packages (constitution); deterministic output (constitution)
**Scale/Scope**: 3 grammar packages (Rust: 134 kinds, TypeScript, Python), ~143 rule entries per grammar

## Complexity Tracking

| Violation | Why Needed | Simpler Alternative Rejected Because |
|-----------|------------|-------------------------------------|
| `metadata.generatedAt` timestamp | Useful for debugging stale templates | **Removed per constitution** — use `grammarSha` only for provenance tracking. Timestamps break deterministic output. |
| `overrides.json` per grammar | Provides field names for ~10-15 under-fielded Rust nodes | Automatic detection alone insufficient — same-kind positional nodes need human-provided names. Overrides file is minimal and validated by codegen. |
| 5 wrap heuristics | Cover all unnamed child patterns in tree-sitter grammars | Fewer heuristics would leave gaps (e.g., can't handle `index_expression` without token-positional). More would be speculative. |
